Summary:

In this assignment, I learned how to create and use a GitHub repository along with using Git on the command line. I created a GitHub account and then created a repository named .dotfiles. This repository is used to store configuration files such as .nanorc and README.md.

I also learned how to configure Git using global settings such as my username and email. In addition, I set up SSH authentication by generating a key pair and adding the public key to my GitHub account. One challenge I faced was setting up SSH keys correctly, I made small mistakes such as incorrect file paths and typos, which caused authentication errors.Another issue I encountered was not seeing my .nanorc file when listing directory contents. I learned that files starting with a dot are hidden in Linux, and I needed to use the “ls -a” command to view them. Fixing these issues helped me understand how SSH connects my local machine to GitHub. This allowed me to securely connect my Linux environment to GitHub without using a password.

After setting up Git and SSH, I created files locally in my .dotfiles directory, including a README.md file and a .nanorc configuration file. Then initialized a Git repository, added the files, committed the changes, and pushed them to GitHub.

Overall, this assignment helped me understand how version control works and how GitHub can be used to store and manage files. I also learned how to troubleshoot issues such as merge conflicts and repository connection problems.
